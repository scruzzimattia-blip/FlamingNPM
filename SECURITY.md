# Sicherheitsrichtlinie

## Unterstuetzte Versionen

Aktuell werden Sicherheitsupdates fuer den letzten stabilen Release-Zweig auf `main` (getaggte Versionen) priorisiert. Aeltere Versionen koennen ohne Fix
bleiben — bitte immer auf die neueste Version aktualisieren.

## Meldung einer Schwachstelle

**Bitte keine** oeffentlichen Issues fuer Sicherheitsprobleme.

1. Schreibe eine **private** Meldung an den Maintainer (E-Mail aus dem GitHub-Profil oder „Security advisories“ auf GitHub — siehe unten).
2. Beschreibe: betroffene Version, Reproduktion, moegliche Auswirkung (ohne
   ausnutzbaren Exploit-Code im Detail oeffentlich zu posten).
3. Erlaube angemessene Zeit fuer Analyse und Fix, bevor Details veroeffentlicht werden.

## GitHub „Private vulnerability reporting“

Empfohlen fuer oeffentliche Repositories:

1. Auf GitHub: **Settings** des Repositories → **Security** → **Code security and analysis**
2. **Private vulnerability reporting** aktivieren.
3. Melder sehen dann den Button **Report a vulnerability** unter **Security**.

So bleiben sensible Details aus Issues und Pull Requests heraus.
